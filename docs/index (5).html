How a request flows
auth → route → compute → store → notify
cognito
Validates JWT and confirms which tenant the request belongs to. The custom:tenant_id is baked into every token at login — nobody can fake it.
api gateway
Single HTTPS entry point. Routes all 4 endpoints to the correct Lambda function regardless of which company is calling.
lambda (node.js)
Decodes tenant_id from the verified JWT — never from request input. Attaches it as a mandatory filter to every database query.
rds postgresql + dynamodb
One shared tasks table. Every query carries WHERE tenant_id = $1. Activity logs written to DynamoDB on every write.
sqs + sns + cloudwatch
Async background processing — task creation triggers SQS, worker Lambda logs to DynamoDB and emails via SNS. CloudWatch monitors everything.
