How a request flows

cognito
Validates the JWT and confirms which tenant the request belongs to before anything else runs.
api gateway
Single entry point for every tenant. Routes to the same Lambda functions regardless of who's calling.
lambda
Extracts tenant_id from the verified token — never from request input — and attaches it to every database call.
rds + dynamodb
One shared table per resource. Every query carries a mandatory WHERE tenant_id filter.
sqs + sns + cloudwatch
Background processing for activity logs and alerts, with full observability across every step above.
The core rule: every tenant shares the same tasks table, the same Lambda code, the same API. Isolation is enforced entirely by filtering every query on tenant_id — extracted from the verified JWT, never trusted from request input.
